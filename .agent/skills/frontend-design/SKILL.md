This skill guides creation of clean, professional, and beautiful frontend interfaces. The goal is solid craftsmanship — interfaces that feel flexible, refined, and genuinely considered, with smooth animations that enhance rather than distract.

The user provides frontend requirements: a component, page, application, or interface to build. They may include context about the purpose, audience, or technical constraints.

## Design Thinking

Before coding, understand the context and commit to a clear, professional aesthetic direction:
- **Purpose**: What problem does this interface solve? Who uses it?
- **Tone**: Professional and clean. Choose from: modern minimal, editorial clean, soft material, airy glassmorphism, structured editorial, neutral luxury, calm functional, confident corporate, quiet premium, warm neutral. The common thread is clarity — every choice should feel deliberate and breathable.
- **Constraints**: Technical requirements (framework, performance, accessibility).
- **Differentiation**: What makes this feel *crafted* rather than generic? Focus on proportions, spacing rhythm, and typographic hierarchy — the hallmarks of professional design.

**CRITICAL**: Prioritize clarity, usability, and visual cohesion. Every element should have a reason to exist. Clean design is not absence of design — it is precision and intention.

Then implement working code (HTML/CSS/JS, React, Vue, etc.) that is:
- Production-grade and functional
- Clean, spacious, and visually trustworthy
- Cohesive with a professional, flexible aesthetic
- Polished in spacing, alignment, and typographic detail

## Frontend Aesthetics Guidelines

Focus on:
- **Typography**: Choose refined, readable fonts with clear hierarchy. Avoid generic system fonts like Arial and Roboto. Opt for quality choices: `Geist`, `Sora`, `DM Sans`, `Plus Jakarta Sans`, `Fraunces`, `Lora`, `Instrument Serif`, `Bricolage Grotesque`. Pair a distinctive display or heading font with a clean, legible body font. Establish a tight typographic scale (e.g. 12/14/16/20/28/40px) and stick to it.
- **Color & Theme**: Use a restrained, professional palette. A neutral base (white, off-white, light gray, deep charcoal, or near-black) with one or two purposeful accent colors. CSS custom properties for all colors. Avoid rainbow palettes — pick a dominant tone and one precise accent. Both light and dark themes are valid; choose intentionally.
- **Motion & Animation**: Include a decent amount of smooth, purposeful animation that enhances the flexible feel without overwhelming. Use:
  - Staggered entrance animations on page load (fade + translate, animation-delay)
  - Smooth hover state transitions (0.2–0.3s ease)
  - Subtle scale or lift effects on interactive elements
  - Fluid layout transitions where applicable
  - Soft skeleton loading states if data is involved
  - CSS transitions preferred; Motion library for React when available
  - Keep animations snappy and elegant — never slow or theatrical
- **Spatial Composition**: Generous, consistent spacing. Strong vertical rhythm. Clear grid alignment. Whitespace is a design element — use it. Avoid clutter; let content breathe. Sections should feel structured and well-proportioned.
- **Backgrounds & Visual Details**: Clean backgrounds with subtle depth. Use:
  - Soft gradient meshes or gentle radial gradients for atmosphere
  - Subtle noise textures or grain overlays for tactile quality
  - Light border treatments (1px with low-opacity) for structure
  - Refined shadows (low opacity, slightly larger blur radius — no harsh drop shadows)
  - Gentle glassmorphism (backdrop-filter: blur) where contextually appropriate
  - Avoid heavy textures, loud patterns, or anything that competes with content

NEVER use:
- Overused generic fonts (Inter, Arial, system-ui as primary faces)
- Harsh, high-contrast multi-color palettes
- Heavy drop shadows or skeuomorphic effects
- Busy backgrounds that fight with content
- Abrupt, non-animated state changes
- Cluttered layouts with insufficient whitespace

Interpret each project with professional care. The UI should feel like it was built by a senior designer who values clarity, restraint, and craft. Every interface should feel **solid, flexible, and quietly beautiful** — the kind of design that earns trust immediately.

**IMPORTANT**: Animations should feel natural and fluid. A well-timed entrance animation and smooth hover states go further than dozens of scattered micro-interactions. Aim for cohesion: the motion system should feel like one language, not a collection of effects.

Remember: clean and professional does not mean boring. Exceptional typography, perfect spacing, a well-chosen accent color, and smooth transitions create interfaces that are quietly extraordinary.
